# 代码安全规范（蓝鲸三大红线）

> 基于 IEG 安全规范，覆盖输入校验、鉴权、数据加密三大高危领域。
> 本规范为横切关注点，适用于所有技术栈和服务类型，与前后端框架规范并列强制执行。

---

## 一、红线总览

| 红线 | 核心要求 | 典型违规 |
|------|---------|---------|
| **红线 1：外部输入未校验** | 外部输入进入高危操作前，必须完成服务端强约束校验（类型/长度/格式/白名单） | 仅前端/网关校验；仅判空/黑名单 |
| **红线 2：敏感接口未鉴权** | 高危能力或敏感资源接口必须同时实施身份认证 + 权限校验 | 内网可达即放通；仅登录不校验权限 |
| **红线 3：敏感数据未加密** | 敏感数据在存储、传输、日志、导出任一环节保护不当即违规 | 硬编码凭证；日志记录敏感字段；URL 携带 token |

## 二、红线 1：外部输入未校验

### 2.1 外部输入范围

以下来源一律视为外部输入，**互信不等于可信**：

| 来源 | 示例 |
|------|------|
| HTTP 请求 | 参数 / Header / Cookie / Body / JSON |
| 文件相关 | 文件名、文件内容 |
| 系统配置 | 环境变量、配置下发内容 |
| 数据库 | DB 可写字段 |
| 服务间调用 | RPC / 回调 / Webhook / MQ / 事件总线 / 任务参数 |

### 2.2 合理校验标准

必须同时满足以下全部条件：

| 条件 | 要求 |
|------|------|
| **服务端校验** | 校验在服务端/执行端完成，且在高危使用点之前 |
| **强约束** | 类型 / 长度 / 格式 / 范围 / 枚举或白名单 / 字符集归一化一致性 |
| **不可绕过** | 不得因编码差异、大小写、分隔符、重复参数、路径归一化差异等被绕过 |
| **不依赖外部假设** | 不得以"内网/互信/网关/前端已校验/仅管理员可用"为免检理由 |

**视为违规的不合理校验：**

| 做法 | 问题 |
|------|------|
| 仅判空 | 未约束类型/长度/格式 |
| 仅黑名单 | 无法枚举所有危险输入 |
| 仅简单正则 | 可能被绕过 |
| 仅前端校验 | 服务端无防护 |
| 仅网关策略 | 绕过网关后无保障 |
| 仅依赖来源 IP | IP 可伪造/变化 |

### 2.3 七类高危操作场景

#### 1.1 外部输入用于命令执行

**适用场景**：shell/exec/subprocess/Runtime.exec/ProcessBuilder/child_process、任务调度执行、脚本调用、运维命令封装

**规范要求**：
- 禁止直接拼接外部输入构造 shell 命令
- 使用参数化 API（如 `exec.Command(cmd, args...)` 而非 `sh -c "cmd "+input`）
- 命令白名单校验，拒绝非法命令

#### 1.2 外部输入进入模板解释

**适用场景**：eval/exec、脚本引擎、动态 import/反射调用、表达式引擎、反序列化入口、模板渲染

**规范要求**：
- 禁止将外部输入直接传入 eval 或等价执行函数
- 使用沙箱隔离脚本执行环境
- 反序列化前校验数据来源与结构

#### 1.3 外部输入影响文件路径

**适用场景**：文件写入/读取/下载、上传落盘、导出落盘、临时文件、移动/重命名、解压/解包（zip/tar）、符号链接攻击

**规范要求**：
- 对路径执行归一化（`filepath.Clean`/`path.resolve`）后校验是否在允许目录内
- 禁止路径穿越（`../` 等）
- 解压时校验每个文件的最终路径，防止 Zip Slip

#### 1.4 外部输入控制请求目标

**适用场景**：HTTP Client 请求（URL/Host/Port 由输入影响）、回调地址、代理转发、跳转地址

**规范要求**：
- 使用白名单校验允许的目标域名/IP 范围
- 禁止将外部输入直接拼接为 URL
- 跳转地址须校验 scheme 和 host，禁止开放跳转

#### 1.5 外部输入参与查询构造

**适用场景**：拼接 SQL/NoSQL 条件、管道、排序字段/表名等结构化查询

**规范要求**：
- 使用参数化查询（Prepared Statement）或 ORM，禁止字符串拼接 SQL
- 动态字段名（排序字段、表名）使用白名单枚举校验
- NoSQL 查询同样禁止外部输入直接注入操作符

#### 1.6 外部输入进入渲染解析

**适用场景**：XML 解析（上传/导入 XML、SOAP、SAML）、HTML/模板渲染、富文本预览、Markdown 渲染、前端危险渲染通道（XSS）

**规范要求**：
- 解析 XML 时禁用外部实体（DTD/XXE），使用安全解析器配置
- 输出到 HTML 时做 HTML 转义，避免存储型/反射型/DOM 型 XSS
- 富文本内容使用白名单过滤器（如 DOMPurify）

#### 1.7 外部输入注入协议输出

**适用场景**：HTTP Header/响应拼接（Response Splitting）、正则/解析器导致 ReDoS、解析歧义绕过校验

**规范要求**：
- 禁止将外部输入直接写入 HTTP 响应头
- 避免使用可能引发 ReDoS 的复杂正则，超时/长度限制前置
- 注意 URL 解码歧义，统一归一化后再校验

## 三、红线 2：敏感接口未鉴权

### 3.1 定义

凡提供高危能力或访问敏感资源的接口，必须在服务端同时实施：

| 要素 | 含义 |
|------|------|
| **身份认证** | 确认"你是谁"—— token 有效性、用户身份 |
| **权限校验** | 确认"你能做什么"—— 角色/租户/资源范围 |

任一缺失即违规。

### 3.2 八类鉴权缺失场景

#### 2.1 接口缺失身份认证

**适用场景**：新增 API、后台管理接口、内部运维接口、RPC/SDK 入口、仅凭"内网可达"放通

**规范要求**：
- 所有对外接口必须校验认证 token（JWT/Session/BKAPI Token）
- "内网可达"不等于已认证，内网接口同样需要认证

#### 2.2 接口缺失权限校验

**适用场景**：仅校验登录不校验角色/租户/项目/资源范围、后台管理操作未做 RBAC/ABAC

**规范要求**：
- 认证通过后还需校验权限（RBAC/ABAC）
- 管理类操作必须验证 `is_superuser` 或具体权限项，不可仅凭登录状态放通

#### 2.3 鉴权依赖不可信要素

**适用场景**：`is_admin` 等前端字段判权、URL 前缀隔离、仅网关策略、仅 IP 白名单

**规范要求**：
- 权限判断必须基于服务端数据（数据库/IAM），禁止依赖请求体中的权限字段
- 路由前缀隔离不能替代鉴权逻辑

#### 2.4 服务间调用缺失鉴权

**适用场景**：服务间 RPC、内部 SDK、代调用、组件联动

**规范要求**：
- 服务间调用使用固定密钥或证书双向认证（mTLS）
- 调用链中携带原始用户身份，在终端服务侧再次校验权限

#### 2.5 资源访问缺失对象鉴权

**适用场景**：仅凭 id/task_id/file_id/cluster_id/app_id 直接读写资源，未校验当前用户是否有权访问该资源

**规范要求**：
- 按资源 ID 访问时，必须同时校验该资源属于当前用户/租户
- 防止水平越权（IDOR）

#### 2.6 回调消息缺失验源

**适用场景**：回调接口、Webhook、消息消费、事件触发入口

**规范要求**：
- 回调接口校验来源签名（HMAC-SHA256 等）
- 事件消费前验证事件来源可信性，拒绝伪造消息

#### 2.7 会话操作缺失 CSRF 防护

**适用场景**：Cookie 会话的敏感操作、未使用 CSRF Token、未校验 Origin/Referer

**规范要求**：
- 使用 Cookie 会话的敏感变更操作必须携带 CSRF Token
- 验证 `Origin`/`Referer` Header，拒绝跨域伪造请求

#### 2.8 调试运维缺失鉴权

**适用场景**：debug/admin/metrics/health 端点、运维开关、临时接口

**规范要求**：
- 所有运维类端点（pprof、prometheus metrics、admin API）必须鉴权或仅监听本地回环
- 临时调试接口上线前必须移除或关闭

## 四、红线 3：敏感数据未加密

### 4.1 敏感数据范围

| 类型 | 示例 |
|------|------|
| 凭证类 | 密码 / Token / AKSK / 连接串 / 私钥 / 加密密钥 |
| 票据类 | 权限票据 / Session |
| 个人信息 | PII（身份证、手机号、邮箱等） |
| 内部信息 | 内部系统地址与关键配置 |

### 4.2 八类加密缺失场景

#### 3.1 敏感信息硬编码暴露

**适用场景**：代码常量/默认配置/示例文件/CI 脚本中写入密码、Token、AK/SK、连接串、私钥、加密密钥

**规范要求**：
- 禁止在代码或配置文件中硬编码任何凭证
- 使用环境变量或密钥管理系统（KMS/Vault）注入
- `.gitignore` 排除含真实凭证的配置文件，CI 中使用 Secret 变量

#### 3.2 敏感信息明文存储

**适用场景**：数据库字段明文存 token/密码/私钥、配置文件明文凭证、缓存/搜索系统写入明文敏感字段

**规范要求**：
- 密码使用强哈希（bcrypt/argon2）存储
- Token/密钥使用加密存储（AES-256-GCM）
- 缓存不存储敏感完整数据，使用 token hash 作为索引

#### 3.3 异常回显泄露细节

**适用场景**：API 返回堆栈/SQL/文件路径、API 返回内部域名/配置内容/密钥片段、异常信息原样透传给调用方

**规范要求**：
- 生产环境错误响应只返回错误码和通用提示，堆栈信息仅写日志
- 统一错误处理中间件，避免底层异常透传

#### 3.4 日志链路泄露敏感

**适用场景**：全量记录 request/response、记录 Authorization/Cookie/Token、trace/span/metrics 写入 PII/凭证

**规范要求**：
- 日志脱敏：Authorization/Cookie/Token 字段替换为 `***`，PII 字段掩码（如 `138****1234`）
- 禁止在 trace/span 中记录凭证或完整 PII
- 日志级别策略：`DEBUG` 日志不进入生产，`INFO` 及以上禁止含原始敏感值

#### 3.5 票据出现在 URL/前端

**适用场景**：token 放 query 参数、跳转携带票据、localStorage/sessionStorage 存长效 token/敏感字段

**规范要求**：
- Token 通过 Authorization Header 传递，禁止放在 URL query string
- 短效 token 可使用 HttpOnly + Secure + SameSite Cookie 存储
- 禁止在 localStorage 存储长效凭证

#### 3.6 敏感传输缺少加密

**适用场景**：HTTP 明文传 token/凭证/PII、跳过证书校验、RPC/MQ 链路未加密传输敏感字段

**规范要求**：
- 所有含敏感数据的接口必须使用 HTTPS/TLS，禁止 HTTP 明文
- 禁止在代码中跳过证书校验（`InsecureSkipVerify: true`）
- RPC 链路使用 TLS 或 mTLS，MQ 启用传输加密

#### 3.7 导出备份暴露敏感

**适用场景**：导出 zip/csv 含凭证/配置/PII、dump/日志包上传对象存储、权限过宽/无过期控制/可公开访问

**规范要求**：
- 导出文件不含凭证，PII 字段脱敏处理
- 上传至对象存储的文件设置私有权限 + 有限期签名 URL（有效期 ≤ 1 小时）
- 备份文件加密存储，访问日志审计

#### 3.8 密码学使用不规范

**适用场景**：自研加密算法、弱算法（MD5/SHA1 用于密码、DES/RC4 等）、固定 IV/盐、同密钥多用途、密钥落配置/日志

**规范要求**：

| 用途 | 推荐算法 | 禁止 |
|------|---------|------|
| 密码哈希 | bcrypt / argon2id | MD5, SHA1, SHA256 直接哈希 |
| 对称加密 | AES-256-GCM | DES, 3DES, RC4, AES-ECB |
| 非对称加密 | RSA-2048+ / ECDSA P-256+ | RSA-1024 |
| 签名 | HMAC-SHA256+ | MD5 HMAC |
| 随机数 | `crypto/rand` | `math/rand` |

- 禁止自研加密算法
- IV/Nonce 每次加密必须随机生成，密钥按用途隔离

## 五、代码评审检查清单

> 执行 Code Review 时，逐项核查以下条目。

### 5.1 输入校验

- [ ] 所有外部输入进入高危操作前，服务端有强约束校验
- [ ] 校验逻辑不依赖前端/网关/IP 等外部假设
- [ ] 无 SQL 字符串拼接，使用参数化查询
- [ ] 文件路径经过归一化并限制在安全目录内
- [ ] 命令执行使用参数化 API，无 shell 拼接
- [ ] XML 解析禁用外部实体（DTD/XXE）
- [ ] 输出到 HTML 有转义，无 XSS 风险

### 5.2 鉴权

- [ ] 所有新增接口有身份认证
- [ ] 权限校验在服务端完成，不依赖请求体字段
- [ ] 按资源 ID 访问时校验资源归属（防 IDOR）
- [ ] 服务间调用有认证机制
- [ ] 回调/Webhook 验证来源签名
- [ ] Cookie 会话的敏感操作有 CSRF 防护
- [ ] 运维/调试端点鉴权或仅本地访问

### 5.3 敏感数据

- [ ] 无硬编码凭证（密码/Token/AK/SK/私钥）
- [ ] 密码使用强哈希存储（bcrypt/argon2id）
- [ ] 日志中敏感字段已脱敏
- [ ] 错误响应不包含堆栈/SQL/内部地址
- [ ] Token 不出现在 URL query string
- [ ] HTTPS/TLS 强制开启，无跳过证书校验
- [ ] 加密算法符合规范（AES-256-GCM / HMAC-SHA256+）
- [ ] IV/Nonce 每次随机生成
